MemUse

The MemUse package is an intentionally over-engineered R package for determining the memory usage of non-allocated, dense, in-core objects. So for example, say you have an mxn matrix of numeric (double precision) values. This should use roughly m*n*8 bytes of memory (with a little overhead for R's additional information stored in the structs; this overhead is effectively 0 for large m*n). Now imagine taking this idea and turning it into several thousand lines of R code full of completely unnecessary OOP and you get the MemUse package.

The package is meant as somewhat of a love letter to other needlessly complex applications, like the Enterprise Fizzbuzz. But it is also born out of the need for having a "conveniently printed" memory usage, meaning in the "best" digits without needing to think about powers of 1024 (or 1000).

The code is 100% written in R, and so it should install easily and be exactly as portable as R itself.

pbdR

The "Programming with Big Data in R" (pbdR) project seeks to elevate the statistical programming language R to supercomputers. We achieve this in part by mapping high-level syntax to low-level, scalable libraries. The project can be used for performance gains even on a laptop, but be honest; doing it on a supercomputer is way cooler.

Presently, I am the lead developer of several pbdR packages:

pbdDEMO

The pbdDEMO package is a lengthy set of package examples and demonstrations for the pbdR project, together with a comprehensive user guide in the form of a textbook-style vignette. Users unfamiliar with the pbdR project, or distributed computing in general, should most definitely start with this package.

pbdDMAT

The pbdDMAT package is a set of high-level wrappers of low-level methods provided by the pbdBASE package for distributed matrix computations. We offer numerous methods, everything from subsetting with `[` to lm.fit().

Code Readme Changelog User Guide Manual

pbdBASE

The pbdBASE package is a set of low-level Fortran routines and wrappers for performing distributed matrix computations. This is intended only for developers; users wishing to elevate their R computations to large distributed platforms should instead look at pbdDEMO and pbdDMAT.
